# 城市建筑互动升级 Demo 设计

## 目标

将现有 Web 3D 工作环境改造成一座可全局查看的工业城市 Demo。玩家从固定角度的鸟瞰视图观察城市，点击六座指定建筑查看信息，并把建筑从 1 级升级到 3 级。首版用于验证城市构图、建筑选择反馈与外观升级，不加入经济系统。

## 参考图

- `example/cityview.jpg`：城市布局参考。保留其道路十字结构、分区关系、右上河流、工业建筑群、停车区域和车辆分布特征。
- `example/citystyle.jpg`：建筑风格参考。采用写实工业街区的灰白屋顶、旧沥青、仓库体块、停车场和少量绿化，但以程序化低多边形方式实现。

参考图用于构图和风格，不要求逐像素或逐建筑复刻。

## 平台与镜头

- 首要目标为桌面横屏浏览器，并保持基本移动端可读性。
- 相机使用约 45 度固定鸟瞰角度，始终维持同一朝向。
- 玩家可在限制范围内平移和缩放。
- 禁止旋转镜头，避免离开已设计的城市构图。
- 初始镜头必须能看见完整城市。
- 平移边界不能让城市完全离开视野；缩放边界需兼顾全局和建筑识别。

## 城市布局

城市以程序化几何体搭建，包含：

- 深灰色主干道与支路。
- 道路交叉口、路缘和停车区域。
- 右上方蓝绿色河流及河岸绿地。
- 六座互动建筑。
- 多座不可互动的仓库、厂房和街区填充建筑。
- 静态车辆、货车、废车堆、油泵、停车位、树木与路灯等识别性道具。

布局遵循 `cityview.jpg` 的相对关系：

- 废车回收站位于城市左上工业地块。
- 物流中心位于上方中部地块。
- Clubhouse 位于上方偏右地块。
- 修车厂位于左侧中部地块。
- 加油站位于下方偏左地块。
- 商业区位于右侧大地块。

## 互动建筑

六座互动建筑为：

1. 废车回收站
2. 物流中心
3. 加油站
4. 修车厂
5. Clubhouse
6. 商业区

每座建筑具有稳定 ID、中文名称、类型、位置、占地尺寸、主色、强调色和三级外观配置。

不可互动建筑不能打开面板，也不能进入升级流程。

## 选择交互

- 鼠标悬停互动建筑时，建筑提亮并显示清晰轮廓或底座高亮。
- 鼠标离开后恢复；已选中建筑继续保持选中反馈。
- 点击互动建筑后，将其设为当前选中建筑并打开右侧信息面板。
- 点击另一座互动建筑时直接切换选择。
- 点击城市空地关闭信息面板。
- 操作面板时事件不得穿透到 3D 场景。
- 建筑交互区域应覆盖完整建筑体块，不能要求点击过小的装饰件。

## 升级流程

- 所有互动建筑初始为 1 级。
- 最高等级为 3 级。
- 信息面板显示建筑名称、当前等级、等级进度和升级按钮。
- 点击升级按钮后，当前建筑立即提升 1 级。
- 建筑等级变化仅影响当前建筑，不影响其他建筑。
- 达到 3 级后，按钮文案显示“已满级”并禁用。
- 本 Demo 不使用金币、升级耗时、前置条件、产出计时或持久化存档。

## 三级外观规则

各建筑升级后必须出现可见差异，不能只改变面板数字：

- 1 级：基础主体与少量类型识别道具。
- 2 级：主体增高或扩建，增加招牌、雨棚、车位、货物或设备。
- 3 级：增加第二段体块、醒目标识、更多附属设施和更强强调色。

不同建筑的识别重点：

- 废车回收站：围栏、废车堆、压缩或吊装设备。
- 物流中心：仓库、装卸口、货箱和货车。
- 加油站：顶棚、油泵和便利店体块。
- 修车厂：车库门、维修位和停放车辆。
- Clubhouse：深色门面、招牌和户外聚集区。
- 商业区：大体量商业建筑、多个店面或屋顶标识。

## UI 设计

- 左上角显示城市标题与简短操作提示。
- 右侧建筑信息面板只在选中建筑时出现。
- 面板采用深色半透明底、浅色文字和暖黄色强调色，与现有工作区视觉语言一致。
- 面板包含关闭按钮，桌面端宽度不遮挡城市主体。
- 小屏幕下面板改为底部抽屉式区域，避免横向挤压。
- 所有按钮提供 hover、active、focus-visible 和 disabled 状态。

## 技术架构

### 数据层

- `src/game/buildingCatalog.ts`：六座互动建筑的静态目录和三级外观参数。
- `src/game/cityLayout.ts`：道路、河流、互动建筑、环境建筑和装饰物坐标。
- `src/game/buildingUpgrade.ts`：纯升级函数与等级上限规则。

### 状态层

Zustand 城市状态保存：

- `selectedBuildingId`
- 六座建筑的当前等级
- `selectBuilding(id)`
- `clearSelection()`
- `upgradeBuilding(id)`
- `reset()`

状态层不保存 hover 状态；hover 由单个建筑组件本地管理。

### 场景层

- `CityScene`：组合城市环境和镜头控制。
- `CityGround`：地面、道路、河流与地块。
- `CityEnvironment`：不可互动建筑、车辆和静态装饰。
- `InteractiveBuilding`：命中区域、hover、选择和等级外观。
- `BuildingModel`：根据建筑类型和等级组合程序化体块。
- `CityCameraControls`：固定角度、禁用旋转并限制平移缩放。

### UI 层

- `CityHud`：标题与操作提示。
- `BuildingPanel`：建筑详情、等级与升级操作。
- 现有 `AppErrorBoundary` 和 Drei `Loader` 继续使用。

## 数据流

1. `CityScene` 从布局数据创建六座互动建筑。
2. `InteractiveBuilding` 根据 ID 从 store 获取自身等级与选中状态。
3. 玩家点击建筑，store 更新 `selectedBuildingId`。
4. `BuildingPanel` 根据选中 ID 读取目录和等级。
5. 玩家点击升级，store 调用纯升级规则。
6. 等级更新触发当前 `BuildingModel` 重新渲染对应外观。
7. 点击空地调用 `clearSelection()`。

## 错误与边界处理

- 未知建筑 ID 不得导致页面崩溃；升级请求应保持状态不变。
- 已达 3 级的建筑再次升级时保持 3 级。
- 面板遇到目录缺失时不渲染错误建筑信息。
- 相机平移和缩放始终受边界限制。
- 不加载外部 3D 模型，降低首版资源失败风险。
- 应用级渲染错误继续由 `AppErrorBoundary` 提供中文回退。

## 测试

自动化测试至少覆盖：

- 建筑目录恰好包含六座指定互动建筑，ID 唯一。
- 所有建筑初始为 1 级。
- 单次升级只影响目标建筑。
- 1 级可升至 2 级，2 级可升至 3 级，3 级保持不变。
- 未知 ID 不改变状态。
- 选择、切换选择和清除选择。
- 面板显示正确建筑与等级。
- 升级按钮更新等级，满级后禁用并显示“已满级”。
- UI 操作不会触发场景清空选择。
- 镜头配置禁止旋转并具有缩放、平移边界。

最终必须通过：

- TypeScript 类型检查
- ESLint
- Prettier 格式检查
- 全套 Vitest 测试
- Vite 生产构建
- 开发服务器 HTTP 200 冒烟

## 验收标准

1. 页面初始显示完整城市鸟瞰图，不再显示旋转立方体。
2. 城市空间关系与 `cityview.jpg` 的主要分区一致。
3. 整体配色和工业建筑语言可识别为 `citystyle.jpg` 所示方向。
4. 六座指定建筑都可悬停、点击、选中和升级。
5. 每座建筑都有三级可见外观变化。
6. 点击空地关闭面板，操作面板不误触场景。
7. 镜头只能平移和缩放，不能旋转，也不能脱离城市边界。
8. 桌面横屏布局完整，小屏幕仍可操作。
9. 自动化检查、测试、构建和 HTTP 冒烟全部通过。

## 非目标

- 不加入金币、资源产出或经济平衡。
- 不加入升级时间、任务、剧情或解锁条件。
- 不加入车辆移动、行人 AI、交通系统或昼夜循环。
- 不加载或制作正式 GLB 模型。
- 不实现后端、账号、联网或存档。
- 不初始化 Git，不配置飞书 CLI。
