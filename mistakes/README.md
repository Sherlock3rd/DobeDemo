# 错题记录

本目录用于记录可复现的部署、运行、格式或协作错误，供后续任务执行前检索复盘。

## 记录模板

新建文件时，文件名建议为 `YYYY-MM-DD-简短描述.md`，并按以下结构填写：

```markdown
# [简短标题]

## 现象

[描述用户或系统表现出的具体问题，包括报错信息、截图说明或复现步骤]

## 原因

[分析根本原因，避免只写表面症状]

## 修复

[列出实际采取的修复步骤或代码变更]

## 防呆

[后续如何避免同类错误再次发生，包括检查清单或流程约束]

## 验证

[如何确认问题已修复，包括命令、预期输出或人工检查项]
```

## 使用方式

- 开始新任务前，检索本目录中与当前任务相关的分类与防呆清单。
- 修复可复现错误后，按模板新增记录，并在 `session/session.md` 变更总账中登记。
