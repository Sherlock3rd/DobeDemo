## 后续演进（现行规则）

本记录描述的是最初的三级外观 Demo。建筑升级系统此后经“独立子建筑经济”演进为当前的“渐进式建筑升级流程”，现行规则以 `docs/superpowers/specs/2026-07-24-progressive-building-upgrade-flow-design.md` 为准，与本文最初范围的差异如下：

- 六类建筑主等级统一为 `1…10`（最高 Lv.10），不再是三级封顶。
- 每座建筑有多个子建筑槽（修车厂 5 槽、其余 10 槽），随主等级逐槽解锁，未解锁槽在面板与 3D 场景中完全隐藏；已解锁 Lv.0 槽仅显示脚手架。
- 详情页只有一个选中项加一个公用子升级按钮，逐级建设；已解锁槽全部追平主等级后进入独立确认页，`确认升级` 才扣费并把主建筑 +1。
- 门槛：修车厂目标 Lv.2–5 无外部门槛、Lv.6–10 受 Clubhouse 约束；其余四类目标 Lv.2–5 受修车厂约束、Lv.6–10 受 Clubhouse 约束；Clubhouse 仅受自身 Lv.40 解锁/进度/资源/上限约束。
- 引入钱、油、物资三资源经济与每 10 秒生产 tick，新账号与重置钱包固定为 `钱 10000 / 油 0 / 物资 0`；建筑战力仅用于展示。
- 城市存档通过 `localStorage` 键 `dobe-city-progression-v1` 持久化（Zustand persist 版本 3），v2→v3 迁移一次性为被隐藏旧子槽退款；帮派声望存于 `gang-progression-v1`。
- 设置面板含“解锁帮派树”“钱/油/物资各 +10000”两个调试动作，及需二次确认的“重置账号”。
- 已初始化并推送 Git 仓库，通过 GitHub Pages 发布。

## 范围

- 六座可互动建筑：废车回收站、物流中心、加油站、修车厂、Clubhouse（保留英文名）、商业区。每座建筑各有互不重叠的合理 footprint、独立配色（主色 + 强调色）。
- 每座建筑三级外观与文字说明（1→2→3 级由简单向复杂演进，例如加油站从双泵岛演进到重型车辆加注区）。点击建筑弹出面板可升级，满 3 级后禁用升级按钮。（后续演进：主等级上限提升至 Lv.10，升级改为逐槽子建筑 + 独立主升级确认页，详见上文「后续演进」。）
- 固定倾角的正交（orthographic）相机：仅支持拖拽平移与滚轮缩放，不支持旋转；平移范围与缩放范围受 `CAMERA_CONFIG` 约束。
- 城市静态场景：草地基底、混凝土地块、道路、河流、环境仓库、静态车辆、树木，均通过轴对齐包围盒（含旋转投影）互不重叠验证。
- 应用外壳：`AppErrorBoundary` 错误边界、Drei `Loader` 加载态、城市 HUD（标题 + 操作提示）、建筑面板（标题、等级 badge、说明、升级/关闭按钮）。
- 响应式桌面横屏控制台视觉：深色工业风背景、HUD 左上角面板、建筑面板桌面态右侧浮层／小屏（<720px）底部抽屉。

## 非目标

- 不实现除六座建筑升级外的具体游戏机制（无经济系统、无多人协作、无存档持久化）。（后续演进：已加入帮派挂机、三资源经济与 `localStorage` 存档持久化，详见上文「后续演进」。）
- 不引入后端、账号系统、数据库或联网功能。
- 不制作正式美术资源（模型、贴图、动画、音频），建筑与场景均为程序化几何体。
- 不支持相机旋转或自由视角。
- 不初始化或提交 Git 仓库。（后续演进：已初始化仓库并推送 `main`，通过 GitHub Pages 发布。）
- 不配置或授权飞书 CLI，不执行飞书文档或电子表格写入。
